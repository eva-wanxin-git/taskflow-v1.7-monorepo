## 💡 为什么必须运行脚本？

### Dashboard状态更新机制

```python
# Dashboard从数据库读取状态
SELECT status FROM tasks WHERE id = '<任务ID>'

# 只有运行脚本才会更新数据库
UPDATE tasks SET status = 'in_progress' WHERE id = '<任务ID>'

# 不运行脚本 = 数据库不更新 = Dashboard看到的还是旧状态
```

### 架构师需要看到进度

```
架构师视角:
- 看Dashboard了解谁在做什么
- 看事件流了解工作进展
- 看状态变化判断是否有问题

如果状态不更新:
- 架构师以为任务没人做
- 可能派发给其他人
- 造成重复工作
```
