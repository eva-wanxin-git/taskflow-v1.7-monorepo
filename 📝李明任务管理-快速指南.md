## 📋 命令速查表

```bash

# 带备注
```

### 提交完成

```bash
# 基础用法
python scripts/李明提交完成.py REQ-001

# 推荐用法（完整信息）
python scripts/李明提交完成.py REQ-001 \
  --hours 4 \
  --summary "实现缓存清除功能，完整测试通过" \
  --files "dashboard.py,templates.py,version_manager.py"
```

### 参数说明

| 参数 | 说明 | 必需 | 示例 |
|------|------|------|------|
| `task_id` | 任务ID | ✅ 是 | REQ-001 |
| `--hours` | 实际工时 | ❌ 否 | 4.5 |
| `--summary` | 完成摘要 | ❌ 否 | "功能已完成" |
| `--files` | 修改文件 | ❌ 否 | "a.py,b.py" |

## 💡 最佳实践

### 接收任务时

✅ **推荐**:
1. Dashboard一键复制提示词
2. 新开Cursor窗口粘贴
3. 立即运行接收脚本更新状态
4. 仔细阅读需求和验收标准

❌ **避免**:
- 不复制提示词就开始开发
- 接收多个任务但不更新状态
- 跳过验收标准检查

---

### 开发过程中

✅ **推荐**:
- 定期查看Dashboard确认任务状态
- 遇到问题及时与架构师沟通
- 遵循代码规范和质量标准
- 编写完整的测试

❌ **避免**:
- 长时间不查看Dashboard
- 偏离原始需求
- 跳过测试环节

---

### 提交完成时

✅ **推荐**:
1. 先确认所有验收标准达成
2. 运行提交脚本（带完整参数）
3. Dashboard复制完成报告
4. 补充详细的技术要点和文件清单
5. 提交给架构师审查

❌ **避免**:
- 功能未完成就提交
- 完成报告信息不完整
- 跳过自测环节

## 🐛 常见问题

### Q1: 点击按钮没有复制？

**原因**: 浏览器需要HTTPS或localhost才能使用Clipboard API

**解决**: 
- ✅ 使用 `localhost` 而不是 `127.0.0.1`
- ✅ 或在Chrome设置中允许不安全来源的剪贴板访问

---

### Q2: 脚本运行失败？

**错误**: `无法连接到Dashboard`

**解决**:
```bash
# 确保Dashboard正在运行
cd taskflow-v1.7-monorepo/apps/dashboard
python start_dashboard.py
```

---

### Q3: 任务状态没更新？

**原因**: Dashboard缓存

**解决**:
1. 点击"🔄 清除缓存"按钮
2. 或按 `F5` 刷新页面

---

### Q4: 找不到任务？

**检查**:
1. 确认任务ID正确（REQ-001, TASK-C1等）
2. 在Dashboard查看任务列表
3. 检查任务状态和分配情况
